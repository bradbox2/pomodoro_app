## 审计后对本项目的修正（两条）

1. **本地捕获 + 本地计时不暂停，照常先做。** 原提案"暂停编辑任务/本地草稿
   标签页方案"修正为：只暂停**云任务浏览/编辑**部分；本地路径（新建草稿、
   选草稿、计时、完成入 Outbox）不依赖任何远端演进，按 Local-first 决策
   立即开发，这是日常主路径。
2. **本项目文档不再记录 GoalSifter 内部设计。** 推进线/焦点计划的实现细节
   （表、页面、交互）以 GoalSifter 项目文档为准；本项目只等待并消费未来的
   "焦点计划快照"契约（见下）。

## 现在就做（不等服务端）

按 Local-first 顺序，接现有数据层（GoalSifterSettings / focus_items /
focus_outbox / GoalSifterClient）：

1. **本地任务计时 UI**（已实现）：默认标签为“本地任务”；单击卡片只设为当前
   任务，再按开始计时，完全离线可用。
2. **"从 GoalSifter 选择"入口**（已实现）：第二标签只读拉现有
   `GET /api/v1/focusflow/tasks` 快照；KR 只显示为远端上下文，隧道不可用不阻塞本地。
3. **显式同步动作**（已实现）：绑定已有 DW / 创建并绑定 DW（422 引导回
   GoalSifter 澄清）/ 手动同步 Outbox（接受或 `duplicate` 后清账）。

4. **本地首页与任务管理收口**（已实现）：首页按控件请求自适应，不再
   固定制造大空白；计时页仍保持桌面顶端的紧凑窗口。新增本地任务管理窗口，
   可重命名、合并或删除空 Project，旧的无意义 Project 可以收敛成少量可维护
   分类，GoalSifter KR 继续只读展示。

以上全部使用**现有已部署契约**（见 ROADMAP 契约小节），无需等待。

## 等服务端就绪后再做（不要提前开发）

GoalSifter 将分期实现焦点计划（周 3 / 日 1+2 的 WIP 投影）；其快照契约
**尚未定稿、尚未部署**。就绪后对方会按契约变更流程发布，本项目届时再开发：

  - "现在"只显示一个任务及计时控制
  - "接下来"最多两个候补

**禁止**：在快照契约发布前，不要基于猜测的字段开发；不要消费全部任务库来

## 本轮保持不变的锁定项

- 一条事件 = 一个完成番茄；`event_id` 幂等；未绑定草稿不上传
- Bearer token 用户手动保存在本机（`%LOCALAPPDATA%\FocusFlow`），
  SSH 只建隧道，绝不读取 VPS 文件
- 提醒不进入 FocusFlow 番茄队列
- 预估 1–4 口径、计分语义见 ROADMAP「客户端计分口径」
