# 2026-04-01 Security Hardening — Credential Externalization & PB Auth Migration

Session topic: Remove hardcoded credentials from source code, migrate PocketBase auth from password to API token, and clarify sync architecture.

---

## 1. Background

A security scan flagged multiple hardcoded secrets in the codebase. Simultaneously, the PocketBase instance on the OpenClaw server (`106.53.153.227`) had its **password-based login disabled**, breaking the existing sync flow.

---

## 2. Sync Impact Analysis

The previous `PBClient._auth()` in `pb_sync_manager.py` called:

```http
POST /api/collections/_superusers/auth-with-password
```

With password auth disabled server-side, this endpoint returns `400 Bad Request`. As a result:
- Token acquisition silently fails
- All sync operations are queued to the offline SQLite queue (`pb_offline_queue.db`)
- **Sync was effectively broken** until this fix

---

## 3. Changes Made

### 3.1 `pb_sync_manager.py` — Config externalization + API token auth

**Before:** hardcoded constants at module level
```python
SSH_HOST  = "ubuntu@106.53.153.227"
PB_EMAIL  = "bot@focusflow.local"
PB_PASSWORD = "FocusFlow2026!pb"
```

**After:** loaded from `.env` via `python-dotenv`
```python
SSH_HOST     = os.environ.get("PB_SSH_HOST", "")
PB_API_TOKEN = os.environ.get("PB_API_TOKEN", "")
PB_EMAIL     = os.environ.get("PB_EMAIL", "")
PB_PASSWORD  = os.environ.get("PB_PASSWORD", "")
```

**`_auth()` logic updated** — API token takes priority:
```python
def _auth(self):
    if PB_API_TOKEN:
        return PB_API_TOKEN          # no HTTP call needed
    # fallback: password auth (if still enabled server-side)
    ...
```

### 3.2 `config.py` — QQ email / password removed

`config.py` was committed to git and contained:
```python
PB_EMAIL    = "649113114@qq.com"     # ← real QQ account
PB_PASSWORD = "njYzZjHb8YJG8mt"     # ← real password
```

Both replaced with `os.environ.get()`. `.env` loading added at module top.

### 3.3 `.gitignore` — `.env` added

`.env` was not previously gitignored. Added:
```
# Environment / Secrets
.env
```

## 5. Required Follow-up (User Action)

To restore sync, create a PocketBase API token:

1. Open PocketBase admin UI on the server: `http://127.0.0.1:18090/_/` (requires SSH tunnel to be active)
2. Navigate to **Settings → API keys → New API key**
3. Copy the generated token
4. Paste into `.env`:
   ```
   PB_API_TOKEN=<paste token here>
   ```

Once set, sync will resume automatically on next app start.
