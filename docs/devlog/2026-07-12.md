## 5. 工程规范化：src/ 包布局重构

按软件工业标准对整个仓库做了一次结构规范化，分两个提交落地。

### 5.1 提交 A（`feat`）— 先落地未提交的成果作为安全基线

- 上一轮完成的本地优先运行时 + GoalSifter DW 绑定此前一直躺在工作区未入库；先原样提交，防止后续重构误删不可恢复。
- 同时移除已死代码：旧云同步栈（`init_pb.py`、`sync_manager.py`、`pb_sync_manager.py`、`sync_history.py`、`tests/test_sync_manual.py`）与废弃的 `.env` / `.env.example`（已无任何源码读取，`test_launch_contract` 亦断言不含 `dotenv`）。

### 5.2 提交 B（`refactor`）— src/ 布局 + 清理

**布局**

- 17 个应用模块全部迁入 `src/focusflow/` 包（用 `git mv` 保留历史）；包内互相引用统一改为绝对导入 `from focusflow.<module> import ...`（共重写 27 个文件）。
- 新增 `src/focusflow/__init__.py`（`__version__`）与 `__main__.py`（支持 `python -m focusflow`）；运行入口包装为 `main()`。
- 一次性维护脚本移入 `scripts/`（`force_gen_dashboard.py`、`migrate_projects.py`），加 `src` 路径引导并把根目录改为向上一级。

**路径修正（重构关键风险点）**

- `main.py` 下沉一层后，`get_base_path()` 与 `AppConfigManager` 的配置回退改为向上两级（`parents[2]`）定位项目根，确保 `images/`、`sound/` 资源与旧数据一次性迁移源仍正确解析。

**打包与工具链**

- 新增 `pyproject.toml`：包元数据、控制台入口 `focusflow`、`pytest` 配置（`pythonpath = ["src"]`）；删除 `pytest.ini` 与重复的根 `conftest.py`（手动/GUI 测试跳过统一到 `tests/conftest.py`）。
- 手写的 `FocusFlow.spec` 纳入版本跟踪（`.gitignore` 加 `!FocusFlow.spec`）：入口改为 `src/focusflow/main.py`、`pathex=['src']`、打包 `focusflow` 一方子模块。
- 两个 `.bat` 启动器改为 `-m focusflow`（`PYTHONPATH=src`），保留 `.venv\Scripts\pythonw.exe` 以满足启动契约测试。

**清理**

- 删除整个 `tools/`（27 个一次性 DB 巡检脚本 + pygame 实验代码，可从 git 历史恢复）、测试 HTML 产物、brainstorm 运行时状态、缓存目录，以及环境变量未展开误建的字面量 `%USERPROFILE%\` 目录。
- `.gitignore` 补全缓存/构建产物规则。

- 无头冒烟测试：18 个包模块全部干净导入；`get_base_path()` 正确指向项目根，`images/`、`sound/` 存在；`main()` 可调用。
- 未实际启动 GUI mainloop（会弹窗/加载音频并阻塞）；本次重构的真实风险（导入路径、资源路径）已由上述方式确认。
