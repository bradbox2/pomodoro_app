# 2026-04-02 Startup Warnings Fix + Legacy SyncManager Retirement + Offline Task Fallback

---

## 1. `python-dotenv` Not Installed — PB Auth Silent Failure

### 问题

启动后出现：
```
WARNING:pb_sync_manager:PB auth: no PB_API_TOKEN and PB_EMAIL/PB_PASSWORD not set
INFO:pb_sync_manager:PB connection: OFFLINE
```

`.env` 中 `PB_EMAIL` 和 `PB_PASSWORD` 明明已填写，但 auth 报未设置。

### 根因

`pb_sync_manager.py` 的 `load_dotenv()` 调用包裹在 `try/except ImportError` 里：

```python
try:
    from dotenv import load_dotenv
    load_dotenv(...)
except ImportError:
    pass  # silently falls through
```

`requirements.txt` 中写了 `python-dotenv>=1.0.0`，但 venv 里实际未安装。
`except ImportError: pass` 静默吞掉异常，env 变量从未被加载，
模块级 `PB_EMAIL = os.environ.get("PB_EMAIL", "")` 读到空字符串。

### 修复

```
pip install python-dotenv
```

### 坑

`try/except ImportError: pass` 是常见的"可选依赖"写法，但当依赖是必需的时候，
静默失败会让问题极难定位。症状是"凭据未设置"而非"模块未找到"，很容易误导排查方向。

## 2. Legacy SyncManager 仍在运行 — 404 警告

### 问题

PB connection OK 后出现：
```
WARNING:SyncManager:PB Auth failed: 404 - {"message":"The requested resource wasn't found."}
WARNING:SyncManager:PB Auth failed: 404 - {"message":"The requested resource wasn't found."}
```

### 根因

`pomodoro_data_manager.py` 和 `sync_history.py` 仍 import 并调用旧的 `SyncManager`
（targets `38.244.21.35:8090`，该实例已搁置）。
`config.py` 的 `PB_SYNC_ENABLED = True` 让这些调用实际发出 HTTP 请求，
旧服务器所有 auth 端点返回 404。

### 修复

`config.py` 中将旧同步开关关闭：

```python
# Legacy SyncManager (38.244.21.35) is retired; new PBSyncManager handles all sync.
PB_SYNC_ENABLED = False
```

同步清理 `.env` 中残留的旧服务器地址：
```
# Before: PB_URL=http://38.244.21.35:8090/api/
# After:  注释说明该字段已弃用
```

### 架构说明

| 路径 | 状态 |
|------|------|
| `PBSyncManager` → SSH tunnel `127.0.0.1:18090` → `106.53.153.227:8090` | **Active** |
| 旧 `SyncManager` → `38.244.21.35:8090` | **Disabled** (`PB_SYNC_ENABLED=False`) |

## 3. 断网时任务列表为空 + 联网恢复后不自动刷新

### 问题

**断网启动：** 任务下拉框完全空白，无法选择任何任务，app 实际上不可用。

**联网恢复：** `_reconnect_loop` 重试成功后，SSH tunnel 通了，offline queue 自动 flush，
但任务列表仍然是空的——用户必须手动点 Sync 按钮才能看到云端任务。

### 根因分析

**断网时任务为空：**

`reset_state_and_ui()` 中的 else 分支：
```python
else:
    self.ui.set_project_list(local_projects)
    self.ui.set_task_list([])   # ← 直接置空，本地 DB 的任务从未展示
```

**联网恢复后不刷新：**

`_try_connect()` 成功后只 flush offline queue，没有通知 UI 层：
```python
if self._connected:
    flushed = self.queue.flush(self.pb)
    # ← 这里没有触发任何 UI 更新
```

`PBSyncManager` 运行在后台线程，无法直接调用 `root.after()`，
也没有机制将"已连接"事件传递给主线程。

### 修复

**1. 断网时填充本地 DB 任务（`main.py`）**

```python
else:
    self.ui.set_project_list(local_projects)
    if local_projects:
        first_local = local_projects[0]
        self.ui.project_var.set(first_local)
        local_tasks = self.data_manager.get_tasks_for_project(first_local)
        self.ui.set_task_list(local_tasks)
    else:
        self.ui.set_task_list([])
```

**2. 联网成功后自动刷新任务列表**

`PBSyncManager.__init__` 新增 `on_connected` 可选回调：

```python
# pb_sync_manager.py
def __init__(self, data_dir: str, on_connected=None):
    ...
    self._on_connected_cb = on_connected

def _try_connect(self):
    ...
    if self._connected:
        ...
        if self._on_connected_cb:
            self._on_connected_cb()   # 通知主线程
```

`main.py` 注册回调，通过 `root.after(0, ...)` 安全切回主线程：

```python
self.pb_sync = PBSyncManager(self.data_dir, on_connected=self._on_pb_connected)

def _on_pb_connected(self) -> None:
    self.root.after(0, self._refresh_cloud_tasks)
```

### 行为变化

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 断网启动 | 任务列表空白，不可用 | 显示本地历史任务，正常可用 |
| 断网启动 → 联网恢复 | 任务列表仍空，需手动 Sync | `_reconnect_loop` 重试成功后自动切换到云端任务 |
| 正常联网启动 | 4s 后刷新（可能 race） | 4s 定时 + 连接成功双重触发，以先到者为准 |

### 坑

`on_connected` 回调在 `_reconnect_loop` 的后台线程中执行，
不能直接操作 Tkinter UI，必须通过 `root.after(0, ...)` 转发到主线程。
